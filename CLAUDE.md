# Go2 Python — Unitree Go2 Robot Controller

## Что это

Этот проект управляет **физическим роботом-собакой Unitree Go2** через WebRTC.
Робот реальный, четвероногий, с камерой, IMU, 12 моторами и батареей.

## КРИТИЧЕСКИ ВАЖНО — БЕЗОПАСНОСТЬ РОБОТА

- **НИКОГДА не врезайся в стены, мебель, людей и животных.** Перед движением проверяй телеметрию (obstacles_avoid, foot_force) и камеру.
- **Перед любым движением убедись**, что робот стоит (`mode != 0`) и батарея > 20%.
- **Не отправляй опасные трюки** (front_flip, handstand, bound) без явного запроса пользователя — робот может упасть и сломаться.
- **Не делай резких движений** — используй скорости 0.1-0.5, не 1.0.
- **Всегда отправляй `stop` после движения** — не оставляй робота двигаться без контроля.
- **Если батарея < 15%** — предупреди пользователя и не выполняй энергозатратные команды.

## CLI — как пользоваться

CLI расположен в `main.py`. Все команды поддерживают `--json` для машинного вывода.

### Базовое подключение
```bash
python main.py                              # интерактивный режим (IP из GO2_IP или 192.168.1.66)
python main.py --ip <ROBOT_IP>              # указать IP явно
python main.py --json ...                   # JSON-вывод для программ
python main.py --json --snap /tmp/go2.jpg move -x 0.5 -d 2   # команда + снимок после
```
IP по умолчанию: `192.168.1.66`. Настраивается через переменную окружения `GO2_IP` или флаг `--ip`.

### Флаг --snap PATH
Добавляет захват кадра с камеры **после** выполнения любой команды (move, exec, set, raw). Кадр сохраняется в PATH. В JSON-выводе добавляется поле `"snap": {"path": "...", "size_bytes": N}`. Это позволяет за одно подключение выполнить действие и увидеть результат.

### Субкоманды

#### Получить телеметрию (состояние робота)
```bash
python main.py --ip <IP> telemetry                    # человекочитаемо
python main.py --ip <IP> --json telemetry              # JSON
python main.py --ip <IP> --json telemetry -s -i 0.2   # потоковая, каждые 0.2 сек
python main.py --ip <IP> --json telemetry -s -n 10     # 10 замеров
```

#### Получить изображение с камеры
```bash
python main.py --ip <IP> image -o frame.jpg                    # сохранить кадр
python main.py --ip <IP> --json image -o -                     # base64 JSON на stdout
python main.py --ip <IP> image -o frames.jpg -s -n 5 -i 0.5   # 5 кадров
```

#### Движение
```bash
python main.py --ip <IP> move -x 0.3 -d 2           # вперёд 0.3 м/с, 2 сек
python main.py --ip <IP> move --forward 0.3 -d 2    # то же, shortcut
python main.py --ip <IP> move --left 0.2 -d 1       # влево
python main.py --ip <IP> move --turn-left 0.5 -d 1  # поворот влево
python main.py --ip <IP> --json move -x 0.3 -d 2    # JSON с состоянием после
```
Параметры: `-x` (вперёд/назад), `-y` (влево/вправо), `--yaw` (поворот), `-d` (длительность сек).

#### Выполнить команды
```bash
python main.py --ip <IP> exec stand_up
python main.py --ip <IP> exec stand_up hello dance1 --delay 2
python main.py --ip <IP> --json exec stand_up
```

#### Установить параметр
```bash
python main.py --ip <IP> set body_height 0.2
python main.py --ip <IP> set speed_level 2
python main.py --ip <IP> set gait 1            # 0=idle, 1=trot, 2=run, 3=stairs
python main.py --ip <IP> set euler 0.1,0,0     # наклон корпуса
```

#### Сырая команда
```bash
python main.py --ip <IP> raw 1008 -p '{"x":0.3,"y":0,"z":0}'
```

#### Список всех команд
```bash
python main.py list
python main.py --json list
```

### Доступные команды (exec)

Основные: `stand_up`, `stand_down`, `sit`, `balance`, `recovery`, `stop`, `damp`
Жесты: `hello`, `stretch`, `content`, `wallow`
Танцы: `dance1`, `dance2`
Трюки (ОСТОРОЖНО): `front_flip`, `front_jump`, `front_pounce`, `wiggle_hips`, `finger_heart`, `handstand`, `cross_step`, `bound`, `moon_walk`
Походки: `economic_gait`, `lead_follow`

### Параметры (set)

- `body_height` — высота корпуса (float, 0.1-0.32)
- `foot_raise_height` — высота подъёма ноги (float)
- `speed_level` — скорость 1-3
- `gait` — тип походки 0-3
- `euler` — наклон r,p,y через запятую
- `video` / `audio` — on/off

### JSON-вывод

С `--json` все команды возвращают JSON со `state` — полное состояние робота:
```json
{
  "status": "ok",
  "state": {
    "battery_soc": 85,
    "position": [0.1, 0.0, 0.32],
    "velocity": [0.0, 0.0, 0.0],
    "imu": {"rpy": [0.01, -0.02, 0.5], ...},
    "foot_force": [100, 95, 102, 98],
    "obstacles_avoid": true,
    ...
  }
}
```

## Механика движения робота

### Оптимальная длительность команд
- **Длительность `-d 2` — оптимум.** Длительности 4–5 секунд ненадёжны — команда может не сработать. Длительность 1 секунда слишком короткая для поворотов.
- **Для поворотов всегда `-d 2`.** Повороты с `-d 1` дают мизерный эффект (~0.1 рад). С `-d 2` поворот реально работает (~0.5 рад при yaw=0.5).
- **Всегда указывай `-d`** — без неё движение практически незаметно.

### Скорость и расстояние
- **Скорость `x=0.5` за 2 секунды даёт ~0.7–0.8 м** реального перемещения.
- **Скорость `x=0.3` за 2 секунды даёт ~0.45 м** — для финального осторожного подхода.
- **Для дистанции ~3 м** нужно 4 шага по 2 секунды при x=0.5 (с коррекциями yaw между ними).
- **Единицы position в телеметрии** примерно соответствуют метрам (position[0] — вперёд, position[1] — влево).

### Дрейф курса (yaw) — КРИТИЧЕСКАЯ ПРОБЛЕМА
- **Робот систематически дрейфует влево.** За каждый 2-секундный шаг (x=0.5) yaw уплывает на +0.15–0.2 рад (~10°).
- **Латеральный дрейф огромный**: за 2.7 м по X робот ушёл на 1.16 м по Y (влево) — это ~23° систематическое отклонение.
- **Коррекция поворотом: ТОЛЬКО `--yaw ±0.5 -d 2`** — это даёт реальные ~0.5 рад (~28°). Команда `--yaw -0.4 -d 1` даёт жалкие ~0.1 рад — бесполезна.
- **Следи за yaw в state.imu.rpy[2]** — корректируй ПОСЛЕ КАЖДОГО шага, если отклонение > 0.1 рад.
- **Добавляй компенсирующий yaw к каждому шагу вперёд**: `move -x 0.5 --yaw -0.15 -d 2` — это частично компенсирует левый дрейф.

### Подъём робота
- **`stand_up` может не сработать с первого раза.** Если робот лежит (body_height < 0.2), используй последовательность: `damp` → (3 сек пауза) → `recovery` → (5 сек пауза) → `stand_up` → (3 сек пауза).
- **body_height > 0.3 = робот стоит.** Поле `mode` может быть 0 даже когда робот стоит — не полагайся на него.
- **foot_force может быть [0,0,0,0]** даже когда робот стоит — тоже ненадёжный индикатор.

### Устойчивость движения
- CLI использует `_sustained_move()`, повторяя команду каждые 100 мс в течение `-d` секунд.
- **speed_level влияет на максимум**: `set speed_level 2` позволяет скорости до ~0.8, `speed_level 3` — до ~1.0.

### Рекомендуемые параметры для навигации
| Задача | Команда | Эффект |
|--------|---------|--------|
| Уверенная ходьба вперёд | `move -x 0.5 --yaw -0.15 -d 2` | ~0.7 м, с компенсацией дрейфа |
| Осторожное приближение | `move -x 0.3 -d 2` | ~0.45 м |
| Быстрый подход | `move -x 0.8 -d 2` (set speed_level 2) | ~1.2 м |
| Поворот вправо | `move --yaw -0.5 -d 2` | ~0.5 рад (~28°) |
| Поворот влево | `move --yaw 0.5 -d 2` | ~0.5 рад (~28°) |
| Обход препятствия | `move -y 0.3 -d 2` | стрейф ~0.4 м |

## Алгоритм навигации к цели

### Принцип: минимум шагов, максимум скорости

**Каждая команда CLI — это ~5-7 секунд** (подключение + выполнение + снимок). Минимизируй количество команд.

### Используй `--snap` для объединения действия и снимка в одну команду

```bash
# Одна команда с --snap (вместо двух отдельных):
python main.py --json --snap /tmp/go2.jpg move -x 0.5 --yaw -0.15 -d 2
```

### Правила скорости — НЕ БУДЬ СЛИШКОМ ОСТОРОЖНЫМ

- **Всегда `x=0.5`** пока цель не займёт >40% высоты кадра (~0.5м до цели)
- **Только последний шаг — `x=0.3`**, когда цель уже крупно в кадре
- **НИКОГДА не делай чистый поворот отдельно** — совмещай поворот с движением вперёд:
  - Цель чуть левее центра → `move -x 0.5 --yaw 0.2 -d 2` (вперёд + поворот)
  - Цель чуть правее центра → `move -x 0.5 --yaw -0.2 -d 2` (вперёд + поворот)
  - Цель сильно сбоку (>30% от края) → `move -x 0.3 --yaw ±0.5 -d 2` (медленнее вперёд + сильный поворот)
  - Чистый поворот (`--yaw ±0.5 -d 2` без -x) — только если цель за пределами кадра или нужен разворот >45°

### Цикл навигации
1. **Начальный снимок**: `--json --snap /tmp/go2.jpg exec stand_up` — встать и оценить обстановку
2. **Прочитать снимок** — оценить расстояние и положение цели в кадре
3. **Проверки**: батарея > 20%, body_height > 0.3 (стоит), путь свободен
4. **Шаг вперёд**: `--json --snap /tmp/go2.jpg move -x 0.5 --yaw <коррекция> -d 2`
   - Коррекция yaw: `-0.15` по умолчанию (компенсация дрейфа), корректировать по положению цели в кадре
5. **Прочитать снимок** — оценить результат:
   - Цель по центру, далеко → ещё шаг `x=0.5` (п.4)
   - Цель смещена → совместить поворот с движением вперёд (см. правила выше)
   - Цель занимает >40% кадра → **один** финальный шаг: `move -x 0.3 -d 2`
   - Цель вплотную → `exec stop`
6. **Повторять** пока не дойдёшь

### Типичная навигация на 3м к видимой цели: 5 команд
1. `exec stand_up` + snap — оценка (1 команда)
2. `move -x 0.5 --yaw -0.15 -d 2` + snap — шаг ~0.7м (1 команда)
3. `move -x 0.5 --yaw <коррекция> -d 2` + snap — шаг ~0.7м (1 команда)
4. `move -x 0.5 --yaw <коррекция> -d 2` + snap — шаг ~0.7м (1 команда)
5. `move -x 0.3 -d 2` + snap → `exec stop` — финальный подход (1-2 команды)

### Важно
- **`x=0.5` — основная скорость**, `x=0.3` — только финальный шаг
- **Совмещай поворот с движением** — не трать отдельную команду на чистый поворот
- **`--snap` экономит время** — одно подключение вместо двух
- **Поворот всегда `-d 2`** — с `-d 1` поворот практически не работает
- **Не использовать `-d` больше 2** — длинные команды ненадёжны
